Lists & List Comprehensions

What are Lists?

Lists are dynamic, ordered collections that can store multiple items of different data types. They are mutable, meaning you can modify them after creation.

üìä List Structure Visualization

[0] ‚Üí 'apple'
[1] ‚Üí 'banana'
[2] ‚Üí 'cherry'
[3] ‚Üí 42
[4] ‚Üí True

Lists use zero-based indexing and can store mixed data types

# Basic List Operations
fruits = ['apple', 'banana', 'cherry']
numbers = [1, 2, 3, 4, 5]
mixed = ['hello', 42, True, 3.14]

# List methods
fruits.append('orange') # Add to end
fruits.insert(1, 'mango') # Insert at index
fruits.remove('banana') # Remove by value
popped = fruits.pop() # Remove and return last

# List Comprehensions
squares = [x**2 for x in range(10)]
even_squares = [x**2 for x in range(10) if x % 2 == 0]
nested = [[i*j for j in range(3)] for i in range(3)]

üöÄ List Comprehensions vs Traditional Loops

List Comprehension: More Pythonic, faster, and more readable for simple operations.

[expression for item in iterable if condition]

‚úÖ Advantages

  • Dynamic size
  • Ordered collection
  • Supports all data types
  • Rich set of methods

‚ùå Considerations

  • Slower than arrays for large datasets
  • More memory overhead
  • Not optimized for mathematical operations

Dictionaries & Sets

Dictionaries: Key-Value Storage

Dictionaries store data in key-value pairs, providing fast lookup times. Keys must be unique and immutable.

Sets: Unique Collections

Sets store unique elements with no duplicates, perfect for membership testing and mathematical operations.

Feature Dictionary Set List
Ordered Yes (Python 3.7+) No Yes
Duplicates Keys: No, Values: Yes No Yes
Mutable Yes Yes Yes
Access Method By key Membership only By index
Use Case Mapping relationships Unique collections Ordered sequences
# Dictionary Examples
student = {
    'name': 'Alice',
    'age': 20,
    'grades': [85, 90, 78]
}

# Dictionary methods
print(student.get('name', 'Unknown')) # Safe access
student.update({'major': 'Computer Science'})
keys = student.keys()
values = student.values()
items = student.items()

# Set Examples
colors = {'red', 'green', 'blue'}
numbers = set([1, 2, 2, 3, 3, 4]) # {1, 2, 3, 4}

# Set operations
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}
union = set1 | set2 # {1, 2, 3, 4, 5, 6}
intersection = set1 & set2 # {3, 4}
difference = set1 - set2 # {1, 2}

üéØ Dictionary vs Set Operations

Dictionary Operations:

  • dict[key] = value
  • dict.get(key, default)
  • dict.pop(key)
  • dict.update(other)

Set Operations:

  • set.add(element)
  • set.union(other)
  • set.intersection(other)
  • set.difference(other)

Exception Handling

What is Exception Handling?

Exception handling is a programming construct that allows you to catch and handle errors gracefully, preventing your program from crashing unexpectedly.

üîÑ Exception Handling Flow

try:
    # Code that might raise an exception
    risky_operation()
except SpecificException:
    # Handle specific exception
    handle_specific_error()
except Exception as e:
    # Handle any other exception
    handle_general_error(e)
else:
    # Executes if no exception occurred
    success_operation()
finally:
    # Always executes
    cleanup_operation()
# Basic Exception Handling
try:
    number = int(input("Enter a number: "))
    result = 10 / number
    print(f"Result: {result}")
except ValueError:
    print("Please enter a valid number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")
except Exception as e:
    print(f"An unexpected error occurred: {e}")
else:
    print("Operation completed successfully!")
finally:
    print("Cleaning up resources...")

# Custom Exceptions
class CustomError(Exception):
    """Custom exception for specific use cases"""
    def __init__(self, message, error_code=None):
        super().__init__(message)
        self.error_code = error_code

# Raising exceptions
def validate_age(age):
    if age < 0:
        raise CustomError("Age cannot be negative", 100)
    if age > 150:
        raise CustomError("Age seems unrealistic", 101)
    return True

Common Built-in Exceptions

  • ValueError: Invalid value
  • TypeError: Wrong type
  • IndexError: Invalid index
  • KeyError: Missing key

File/IO Exceptions

  • FileNotFoundError: File doesn't exist
  • PermissionError: Access denied
  • IOError: Input/Output error

üéØ Best Practices for Exception Handling

  • Be specific with exception types - catch the most specific exception first
  • Don't use bare except: clauses
  • Use finally for cleanup operations
  • Log exceptions for debugging purposes
  • Create custom exceptions for domain-specific errors

Object-Oriented Programming

What is Object-Oriented Programming?

OOP is a programming paradigm based on the concept of "objects" that contain data (attributes) and code (methods). It provides a way to structure programs to make them more modular and reusable.

üèõÔ∏è OOP Pillars

üîí Encapsulation

Bundling data with methods

üß¨ Inheritance

Creating new classes from existing ones

üé≠ Polymorphism

Same interface, different implementations

üéØ Abstraction

Hiding complex implementation details

# Basic Class Definition
class Student:
    # Class variable (shared by all instances)
    school = "University of Python"

    def __init__(self, name, age, student_id):
        # Instance variables (unique to each instance)
        self.name = name
        self.age = age
        self.student_id = student_id
        self.__grades = [] # Private attribute

    def add_grade(self, grade):
        """Public method to add a grade"""
        if 0 <= grade <= 100:
            self.__grades.append(grade)
        else:
            raise ValueError("Grade must be between 0 and 100")

    @property
    def gpa(self):
        """Calculate GPA from grades"""
        if not self.__grades:
            return 0.0
        return sum(self.__grades) / len(self.__grades)

    def __str__(self):
        return f"Student: {self.name} (ID: {self.student_id})"
